# Benutzerdefinierte Klassen erstellen

Die in i-doit up enthaltenen Klassen decken den Großteil der Anforderungen an die IT-Dokumentation ab.
Wenn im Standardmodell ein Konzept fehlt, kann ein Administrator eine **benutzerdefinierte Klasse** definieren: einen neuen Objekttyp mit eigenem Symbol, eigenen zugeordneten Kategorien und eigenen Sammlungszugehörigkeiten.

## Rechte

Zum Erstellen oder Bearbeiten einer Klasse ist die Berechtigung *Klassen verwalten* unter *Verwaltung* erforderlich.
Siehe [Rechte und Berechtigungen](rights-and-permissions.md).

## Den Klasseneditor öffnen

1. Öffnen Sie **Einstellungen** über das Benutzermenü (Avatar oben rechts).
2. Gehen Sie zu **CMDB-Konfiguration ▸ Klassen**.
3. Klicken Sie oberhalb der Liste auf **Neue Klasse +**.

## Kursdetails

Im Klassenformular werden folgende Angaben abgefragt:

| Feld | Anmerkungen |
|---|---|
| **Name** | Der in der Finder-Seitenleiste, im Dropdown-Menü „Alle Klassen“ und in der Kopfzeile der Detailseite jedes Objekts angezeigte Name. |
| **Symbol** | Wählen Sie aus einem vordefinierten Symbolsatz. Das Symbol wird überall neben dem Klassennamen angezeigt. |
| **Sammlungen** | Eine oder mehrere [Sammlungen](../user/basics/collections.md), zu denen die Klasse gehört. |
| **Kategorien** | Die Menge der [Kategorien](../user/basics/categories-and-attributes.md), die der Klasse zugeordnet sind. Fügen Sie integrierte oder [benutzerdefinierte Kategorien](custom-categories.md) hinzu. |

Speichern, um die Klasse zu erstellen.
Neue Klassen werden sofort im Finder angezeigt.

## Bearbeiten und löschen

Öffnen Sie eine vorhandene Klasse aus der Liste, um ihr Symbol, ihre Sammlungszugehörigkeiten oder ihre Kategorieneinstellungen zu ändern.
Informationen zum Löschvorgang und zu den Auswirkungen, wenn noch Objekte dieser Klasse vorhanden sind, finden Sie unter [Klassen und Sammlungen verwalten](class-collection-management.md).
