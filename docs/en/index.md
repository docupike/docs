# i-doit up documentation

Welcome to the locally-built i-doit up documentation.
This is a working copy used for review before the content is published to the public docs site.

## What's inside

The pages on the left are grouped by audience:

- **User guide, Basics** explains how to work with objects, classes, locations, and the things every user touches: favorites, IPs, networks.
- **User guide, Finder** covers the table-shaped browser at the centre of the application: column presets, saved searches, filters, bulk actions.
- **User guide, Reporting** describes how to save and re-run reports across the CMDB.
- **Admin guide** collects everything that needs administrative rights: tenants, classes/categories management, the user-management area, and the add-ons surface.

## How to read this site

Each page is self-contained.
Follow the *Further readings* list at the bottom of every page to jump to related topics.

If you spot something that does not match the live application, please flag the page so the next pass can correct it.
